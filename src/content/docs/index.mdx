**ee** is a CLI tool that brings structure and validation to environment variable management. Define schemas, organize configurations across environments, and deploy with confidence knowing your variables are validated and consistent.


		<div class="mt-4">
      <div class="rounded-lg flex flex-col">
        <p class="text-lg font-medium mb-3">Apply Environments</p>
        <p class="leading-relaxed">Apply environment configurations to your shell or run commands with specific variable sets. Load from .env files or project environments defined in your .ee file.</p>
      </div>
    </div>
		<div class="m-auto w-full">
			```bash
			# Apply .env file
			ee apply .env

			# Apply project environment (from .ee file)
			ee apply production

			# Run commands with environment
			ee apply production -- npm start
			ee apply development -- npm test
      ```
    </div>

  </div>
  <div class="grid grid-cols-1 lg:grid-cols-2 gap-8 items-start">
			<div class="m-auto w-full">
      ```bash
      # Create a schema
      ee schema create web-service \
        --description "Schema for web services" \
        --variable "PORT:number:Server port:false:8080" \
        --variable "DEBUG:boolean:Debug mode:false:false"

      # Initialize project with schema
      ee init --schema web-service

      # Environments defined in .ee file:
      # { "environments": {
      #     "development": { "env": ".env.development" },
      #     "staging": { "env": ".env.staging" }
      # }}
      ```
    </div>

		<div class="mt-4">
      <div class="rounded-lg flex flex-col">
        <p class="text-lg font-medium mb-3">Initialize Projects</p>
        <p class="leading-relaxed">Initialize projects with .ee configuration files. Define environments and their .env file sources directly in the .ee file for a portable, version-controlled setup.</p>
      </div>
    </div>

  </div>
</div>

## Features

<CardGrid>
	<Card title="Schema Validation" icon="approve-check">
		Define variable types, patterns, and requirements. Catch configuration errors before they reach production.
	</Card>
	<Card title="Multi-Environment Support" icon="setting">
		Manage development, staging, and production configurations from a single tool.
	</Card>
	<Card title="Variable Inheritance" icon="puzzle">
		Share common configuration across environments while maintaining environment-specific overrides.
	</Card>
	<Card title="Multiple Export Formats" icon="document">
		Export to .env, JSON, YAML, or load directly into your shell environment.
	</Card>
</CardGrid>

<div class="space-y-16 my-12">
  <div class="grid grid-cols-1 lg:grid-cols-2 gap-8 items-start">
		<div class="mt-4">
      <div class="rounded-lg flex flex-col">
        <p class="text-lg font-medium mb-3">Push secrets to GitHub & Cloudflare</p>
        <p class="leading-relaxed">Push your environment secrets directly to GitHub or Cloudflare Workers. Configure origins in your .ee file and push secrets from your local .env files using the tools you already have.</p>
      </div>
    </div>
		<div class="m-auto w-full">
			```bash
			# Configure origins in .ee file
			# { "origins": { "github": { "type": "github" } } }

			# Push production secrets to GitHub
			ee push github production

			# Preview what would be pushed
			ee push production --dry-run
      ```
    </div>

  </div>
  <div class="grid grid-cols-1 lg:grid-cols-2 gap-8 items-start">
			<div class="m-auto w-full">
```bash
# Export environments
ee apply production --dry-run --format dotenv > .env.prod
ee apply production --dry-run --format json > config.json

# Apply .env files directly
ee apply .env.staging

# Filter current environment variables
ee --filter 'NODE*,PORT' --format json
```
    </div>

		<div class="mt-4">
      <div class="rounded-lg flex flex-col">
        <p class="text-lg font-medium mb-3">Export and import</p>
        <p class="leading-relaxed">Export configurations to various formats including .env files and JSON. Apply existing .env files directly, and filter current environment variables with patterns.</p>
      </div>
    </div>

  </div>
</div>
